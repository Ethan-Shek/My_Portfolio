Project Overview

Project Overview

Fruit Slice is a fast-paced arcade game inspired by Fruit Ninja, reimagined to emphasize responsive interaction and clean visual communication. The project challenged me to blend intuitive user experience design with physics-based gameplay, creating smooth and rewarding player feedback loops.

Design Approach

Design Approach

✦ Gameplay Flow: Built a swipe-based interaction system using raycasting and mouse position tracking to simulate slicing motion. This interaction was tuned for precision and immediate feedback.
✦ Visual & Audio Feedback: Added satisfying slice and explosion sound effects, along with Japanese-inspired background music, to elevate immersion and rhythm. Similar to micro-interactions in UI design.
✦ Spawning Logic: Developed a randomized object launcher using Perlin and Gaussian randomness for natural motion. This approach created smooth, non-repetitive gameplay similar to procedural UX animation.
✦ Interface Design: Designed a clear HUD with live score and life tracking. The minimal layout and hierarchy mirror digital product dashboards, ensuring that key info is always visible without distraction.
✦ Game States: Players earn +1 point for slicing fruit and lose a life for hitting a bomb. Once all lives are lost, the game transitions smoothly to a game-over state.

Core Scripts

Core Scripts

✦ Destroyer Script: Handles swipe detection and object destruction logic using Physics2D.Raycast for accuracy.
✦ RandomSpawner Script: Controls timing, angle variation, and object selection for randomized launches.
